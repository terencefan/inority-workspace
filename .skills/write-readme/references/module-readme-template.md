# 模块 README 模板

适用于子目录、包级模块或边界清晰的功能组件。

```md
# <模块名> 模块简介

> 用一句话说明这个模块在整体系统里的职责和边界。

## 模块简介

说明模块的定位、用途和典型调用者。

## 职责边界

列出模块负责的内容，以及明确不负责的内容。

## 入口与公共接口

说明主入口文件、导出对象、CLI、API 或扩展点。

## 依赖关系

说明它依赖哪些内部模块、外部服务或共享组件。

## 扩展方式

说明如果要新增能力，应该从哪里接入，遵循什么约束。

## 相关文件

| 路径 | 说明 |
|------|------|
| `<路径>` | `<职责说明>` |
```
