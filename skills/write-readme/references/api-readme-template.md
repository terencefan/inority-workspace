# API README 模板

适用于 REST、GraphQL、RPC、SDK 或接口聚合文档。

```md
# <接口或模块名> README

> 用一句话说明这个接口边界、主要调用场景和使用前提。

## 概述

说明接口负责什么，不负责什么，以及典型调用者是谁。

## 访问入口

列出域名、路径前缀、环境差异或代理改写规则。

## 认证

说明认证方式、凭证来源和鉴权注意事项。

## 响应约定

说明统一响应包裹、错误格式或状态码约定。

## 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/path` | `<接口用途>` |

## 接口详情

### `<方法> <路径>`

- 用途：<说明接口做什么>
- 请求参数：<列关键参数或链接到更详细定义>
- 响应示例：<给出真实或接近真实的示例>
- 注意事项：<列兼容性、限流、缓存或错误语义>

## 调试与联调建议

列出 curl、HTTPie、Postman、日志位置或本地代理方式。

## 代码位置

| 路径 | 说明 |
|------|------|
| `<路径>` | `<职责说明>` |
```

优先使用：

- 接口总览表
- 真实请求/响应示例
- 明确的路径前缀
- 已确认的标准链接或协议约定
